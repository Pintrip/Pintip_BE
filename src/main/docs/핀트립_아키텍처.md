## 1. 왜 세션인가

핀트립 MVP는 회원가입/로그인 없이 동작한다. 중요한 것은 **사용자 계정**이 아니라 **오늘의 여행 상태**다.

- 새로고침·외부 링크 복귀 후에도 상태 유지
- 서버에서 후기·완료 여부 검증
- 해커톤·MVP에서 디버깅 단순화

```text
TripSession = 오늘의 핀트립 상태 (UUID + DB)
```

---

## 2. 세션 방식: UUID + DB

| 방식 | MVP 적합도 |
|------|------------|
| localStorage only | 서버 검증 불가 |
| Cookie 서버 세션 | CORS·외부 앱 이동 이슈 |
| **UUID + DB** | 채택 |

---

## 3. sessionId 흐름

```text
POST /trip-sessions → sessionId(UUID)
프론트: localStorage.setItem("tripSessionId", sessionId)
이후: /trip-sessions/{sessionId}/... 또는 헤더 X-Session-Id
```

만료: 생성 시각 + **2일** (`TripSessionPolicy.EXPIRE_DAYS`)

---

## 4. 상태 흐름 (현재 구현)

```text
동네 목록 → (프론트 랜덤) → 동네 확정
이미지 카드·퀘스트 목록 조회 → (프론트 랜덤) → 세션 생성
퀘스트 수행 → 퀘스트별 후기 저장 (최대 3, quests[].isCompleted)
status → COMPLETED: (1) 후기 3건 (2) 생성 후 2일 경과 (3) PATCH /complete 수동
COMPLETED 세션은 조회 가능, 후기 쓰기는 410
```

랜덤은 **프론트** 책임. 백엔드는 CSV 기반 **고정** 카드·퀘스트 데이터를 제공한다.

---

## 5. 저장 위치

| 데이터 | 저장 |
|--------|------|
| 동네·카드·퀘스트 | DB (`data.sql`) |
| TripSession | DB |
| 퀘스트 후기 | DB (`trip_session_quest_reviews`) |
| sessionId | 프론트 localStorage |
| 이미지 파일 | 프론트 정적 자산 (`imageFile`은 파일명만) |

---

## 6. 스택 (MVP)

```text
Frontend: React, Vercel (/backend proxy)
Backend: Spring Boot REST
DB: MySQL (로컬 Docker / prod RDS)
```
