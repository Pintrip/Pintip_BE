# 1. 서비스 개요

## 서비스명

**핀트립**

## 한 줄 정의

핀트립은 사용자가 선택한 지역에서 랜덤 장소와 랜덤 퀘스트를 뽑아, 평범한 장소를 오늘만의 비주류 여행지로 만드는 웹 서비스다.

## 핵심 콘셉트

기존 여행 서비스는 유명 관광지, 인기 맛집, 리뷰 많은 장소를 중심으로 여행지를 추천한다.

핀트립은 반대로 사용자가 검색하지 않은 평범한 장소에 랜덤성과 퀘스트를 부여해, 그 장소를 새로운 여행 목적지로 재해석한다.

즉, 핀트립의 핵심은  
**“유명한 곳을 찾는 여행”이 아니라 “평범한 곳을 여행지로 만드는 경험”이다.**

---

# 2. 문제 정의

대부분의 여행 서비스는 이미 유명한 장소를 중심으로 작동한다.

- 리뷰가 많은 장소
- 인기 맛집
- 유명 관광지
- SNS에서 자주 보이는 핫플
- 검색 알고리즘에서 상단에 노출되는 장소

그 결과, 특정한 관점으로 보면 충분히 매력적일 수 있는 골목, 작은 공원, 오래된 상가, 시장 입구, 산책로 같은 평범한 장소들은 여행지로 발견되기 어렵다.

---

# 3. 비주류 정의

핀트립에서 비주류 여행지는 단순히 유명하지 않은 장소를 의미하지 않는다.

> 비주류 여행지란 원래 관광지로 소비되지 않지만, 랜덤 퀘스트와 개인의 관점이 부여되었을 때 여행 목적지가 될 수 있는 평범한 장소다.

## 예시

| 평범한 장소 | 랜덤 퀘스트              | 비주류 여행지화      |
|--------|---------------------|---------------|
| 동네 골목  | 가장 오래된 간판 찾기        | 레트로 흔적 탐방지    |
| 작은 공원  | 가장 조용한 벤치 찾기        | 혼자 머무는 쉼표 여행지 |
| 시장 입구  | 처음 보는 가게 이름 기록하기    | 로컬 생활감 관찰지    |
| 하천 산책로 | 물소리가 가장 잘 들리는 지점 찾기 | 소리 수집 여행지     |
| 오래된 상가 | 사라질 것 같은 공간 기록하기    | 도시 흔적 수집지     |

---

# 4. 핵심 경험

핀트립은 웹 기반 서비스이므로 앱처럼 계속 켜두고 위치를 추적하는 방식은 제외한다.

대신 여행 시작 전과 여행 중간의 경험을 나누어 설계한다.

---

## A. 여행 시작 전

사용자가 웹에서 지역을 선택하고 랜덤 장소와 랜덤 퀘스트를 뽑는다.

- 예시 구성:
    - 5개 지역
    - 10개 미션
    - 총 50개 목데이터

이때 **오늘의 비주류 여행 미션 카드**가 생성된다.

---

## B. 여행 중간

사용자는 이동 중 웹을 다시 열어 현재 미션을 확인하거나, 새로운 미션을 추가로 뽑을 수 있다.

도착한 장소에서 간단한 기록을 남기면, 그 기록이 오늘의 핀트립 로그에 쌓인다.

---

## 핵심 플로우

```text
랜딩
→ 지역 선택
→ 랜덤 핀 던지기
→ 랜덤 장소 결정
→ 랜덤 퀘스트 결정
→ 오늘의 미션 카드 생성
→ 지도 링크로 이동
→ 여행 중 기록 추가
→ 퀘스트 완료 체크
→ 여행 후기 작성
→ 핀트립 완료 화면 확인
```

# 5. MVP 범위

## Must Have

| 구분 | 기능        | 설명                         |
|----|-----------|----------------------------|
| 진입 | 랜딩 화면     | 서비스 소개와 시작 버튼 제공           |
| 선택 | 지역 선택     | 사용자가 여행할 지역 선택             |
| 랜덤 | 랜덤 장소 추첨  | 선택한 지역의 장소 후보 중 1개 랜덤 선택   |
| 랜덤 | 랜덤 퀘스트 추첨 | 장소를 비주류 여행지화할 퀘스트 랜덤 선택    |
| 결과 | 미션 카드 생성  | 장소 + 퀘스트 + 기록 가이드 출력       |
| 이동 | 지도 링크 연결  | 네이버/카카오 지도 검색 링크 제공        |
| 기록 | 짧은 기록 등록  | 여행 중 발견한 것, 메모 등을 기록       |
| 결과 | 핀트립 로그 확인 | 오늘 뽑은 장소/퀘스트/기록을 한 화면에서 확인 |

# 7. 개발 우선순위 요약

## P0: 반드시 구현

```text
A101 서비스 소개 확인
```
