# 핀트립 백엔드 폴더 구조

실행 가이드·API: [백엔드_개발_실행계획.md](백엔드_개발_실행계획.md)

---

## 1. 원칙

- 도메인 단위 패키지 + `controller` / `service` / `repository` / `entity` / `dto`
- `entity`를 API 응답으로 직접 반환하지 않음 (DTO 사용)
- `/trip-sessions` 관련 API는 `TripSessionController`에 모음

---

## 2. 패키지 트리 (현재)

```text
src/main/java/pintrip/
├── PintripApplication.java
├── global/
│   ├── config/          # WebConfig(CORS), OpenApiConfig
│   ├── error/           # ErrorCode, BusinessException, GlobalExceptionHandler
│   └── health/          # HealthController
├── domain/dong/
│   ├── controller/      # DongController
│   ├── service/         # DongService, DongContentService
│   ├── repository/
│   ├── entity/
│   └── dto/
├── domain/image/
│   ├── entity/          # DongImageMapping, ImageCardQuest
│   └── repository/
└── domain/session/
    ├── TripSessionPolicy.java
    ├── controller/      # TripSessionController
    ├── service/         # TripSessionService, TripSessionQuestReviewService
    ├── repository/
    ├── entity/          # TripSession, TripSessionQuestReview
    ├── dto/
    └── converter/       # StringListJsonConverter (moodTags)

src/main/resources/
├── application.yaml
├── application-local.yaml
├── application-prod.yaml
└── db/
    ├── schema.sql
    ├── data.sql
    └── README.md

src/main/docs/             # 본 문서들
```

> `pintrip/demo/**` — 초기 스캐폴드. 운영 코드는 `pintrip.domain.*` 기준.

---

## 3. API ↔ 패키지

| API | 컨트롤러 | 서비스 |
|-----|----------|--------|
| `GET /dongs` | `DongController` | `DongService` |
| `GET /dongs/{dongId}/image-cards` | `DongController` | `DongContentService` |
| `POST /trip-sessions` | `TripSessionController` | `TripSessionService` |
| `GET /trip-sessions/...` | `TripSessionController` | `TripSessionService` |
| `PATCH .../complete` | `TripSessionController` | `TripSessionService` |
| `PUT/GET .../quest-reviews` | `TripSessionController` | `TripSessionQuestReviewService` |
