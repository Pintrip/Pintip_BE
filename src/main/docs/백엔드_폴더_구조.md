# 핀트립 백엔드 폴더 구조 (권장)

해커톤 1일, 백엔드 2인 분업 기준 권장 구조다.  
실행 가이드(API·분업·일정)는 `백엔드_개발_실행계획.md`를 본다.

---

## 1. 패키지 원칙

- **도메인 단위 패키지** + 레이어(`controller` / `service` / `repository` / `entity` / `dto`)
- URL prefix `/trip-sessions` API는 `TripSessionController` 한 곳에 모으고, 내부는 각 `Service`에 위임
- `entity`를 API 응답으로 직접 반환하지 않는다 (DTO 사용)

> 부트스트랩: `pintrip.demo` → 당일 `pintrip` 루트로 정리 권장

## 3. API ↔ 패키지

| API | 담당 | 패키지 |
|-----|------|--------|
| `POST /trip-sessions` | BE-1 | `domain/session` |
| `GET /trip-sessions/{sessionId}` | BE-1 | `domain/session` |
| `PATCH /trip-sessions/{sessionId}/region` | BE-1 | `domain/session` |
| `GET /regions` | BE-2 | `domain/region` |
| `POST .../place/random` | BE-2 | `domain/place` → session controller 위임 |
| `POST .../quest/random` | BE-2 | `domain/quest` → session controller 위임 |
| `POST .../logs` | BE-2 | `domain/log` → session controller 위임 |

---

## 4. 충돌 방지 (수정 경로)

| 개발자 | 주로 수정 |
|--------|-----------|
| BE-1 | `global/**`, `domain/session/**`, `domain/log/entity`, `domain/log/repository`, `schema.sql` |
| BE-2 | `domain/region/**`, `domain/place/**`, `domain/quest/**`, `domain/log/service`, `data.sql` |

- `TripSessionController`는 **BE-1 소유**. BE-2는 service만 PR한다.
- `global/error`는 BE-1이 먼저 생성한다.
